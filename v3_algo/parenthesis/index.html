About Parenthesis Checking

Algorithm

  1. Initialize an empty stack.
  2. Scan the expression from left to right.
  3. If the current character is an opening bracket ((, {, or [), push it onto the stack.
  4. If the current character is a closing bracket (), }, or ]), check if the stack is empty:
    • If the stack is empty, the expression is invalid.
    • If the stack is not empty, pop the top element from the stack and check if it matches the current closing bracket:
      • If they match (e.g., ( and )), continue to the next character.
      • If they don't match (e.g., ( and ]), the expression is invalid.
  5. After scanning the entire expression, if the stack is empty, the expression is valid. Otherwise, it's invalid.

Time and Space Complexity

  • Time Complexity: O(n), where n is the length of the expression
  • Space Complexity: O(n) in the worst case when all characters are opening brackets

Examples

Valid Expressions
  • () - Simple matching pair
  • [] - Simple matching pair
  • {} - Simple matching pair
  • ()[]{} - Multiple pairs in sequence
  • {[()]} - Nested pairs
Invalid Expressions
  • (] - Mismatched brackets
  • ([)] - Incorrectly nested
  • {[} - Incomplete closing
  • ) - Only closing bracket
  • ((( - Only opening brackets
