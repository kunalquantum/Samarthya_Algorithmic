About Maximum Depth of Binary Tree

Recursive Approach

  • The maximum depth of a binary tree is the maximum of:
    • The depth of the left subtree + 1
    • The depth of the right subtree + 1
  • Base cases:
    • If the node is null, return 0
    • If the node is a leaf (no children), return 1
  • The algorithm uses a recursive depth-first search (DFS) approach

Time and Space Complexity

  • Time Complexity: O(n) - where n is the number of nodes in the tree
  • Space Complexity: O(h) - where h is the height of the tree (due to recursion stack)
  • Best Case: O(log n) space for a balanced tree
  • Worst Case: O(n) space for a skewed tree

Example

    3
   / \
  9  20
     / \
    15  7

Maximum Depth = 3

  • Root to leaf path through left child: 3 → 9 (depth = 2)
  • Root to leaf path through right child: 3 → 20 → 15 or 3 → 20 → 7 (depth = 3)
  • Therefore, the maximum depth is 3
