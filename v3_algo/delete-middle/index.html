About Deleting Middle Element from Stack

Algorithm (Recursive Approach)

  1. Find the size of the stack.
  2. Calculate the middle position (size/2 + 1 from bottom or size/2 from top for odd size, exactly in the middle for even size).
  3. Use recursion to reach the middle element:
    • Pop elements from the stack and store them in a function call stack (using recursion).
    • When the middle element is reached, skip it (don't push it back).
    • While returning from recursion, push all other elements back to the stack.

Time and Space Complexity

  • Time Complexity: O(n), where n is the number of elements in the stack
  • Space Complexity: O(n) for the recursive call stack

Why Use Recursion?

Stacks only allow operations at one end (the top). To access the middle element, we need to remove elements until we reach the middle. Recursion provides a clean way to remember these elements and put them back in the correct order after removing the middle element.

The recursive approach uses the function call stack to temporarily store the popped elements, which is more elegant than using an additional data structure.

Examples

Example 1: Odd number of elements

Input Stack: [1, 2, 3, 4, 5] (top is rightmost)

Middle Element: 3 (index 2)

Output Stack: [1, 2, 4, 5]

1. Pop and store: 5, 4, 3 (middle - skip this)

2. Push back: 4, 5

3. Result: [1, 2, 4, 5]

Example 2: Even number of elements

Input Stack: [1, 2, 3, 4] (top is rightmost)

Middle Element: 2 (index 1)

Output Stack: [1, 3, 4]

1. Pop and store: 4, 3, 2 (middle - skip this)

2. Push back: 3, 4

3. Result: [1, 3, 4]
