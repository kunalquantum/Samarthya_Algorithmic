About Edit Distance

Dynamic Programming Solution

Calculates the minimum number of operations required to transform one string into another.

Available Operations:

  • Insert: Add a character (Cost: 1)
  • Delete: Remove a character (Cost: 1)
  • Replace: Change one character to another (Cost: 1)

Algorithm Details:

  • Time Complexity: O(m × n)
    • m: length of source string
    • n: length of target string
    • Each cell requires constant time operations
  • Space Complexity: O(m × n)
    • Stores DP table of size (m+1) × (n+1)
    • Additional space for backtracking

Key Steps:

  • Table Construction:
    • Initialize first row and column
    • Compare characters and calculate minimum operations
    • Choose minimum cost operation at each step
  • Backtracking:
    • Start from bottom-right corner
    • Determine operation type at each step
    • Build transformation sequence

Example:

Source: "kitten"

Target: "sitting"

Edit Distance: 3

Operations:

  1. Replace 'k' with 's'
  2. Replace 'e' with 'i'
  3. Insert 'g' at the end
