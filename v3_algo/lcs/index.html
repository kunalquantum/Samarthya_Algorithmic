About Longest Common Subsequence

Dynamic Programming Solution

Finds the longest subsequence present in both sequences in the same relative order.

Algorithm Details:

  • Time Complexity: O(m × n)
    • m: length of first sequence
    • n: length of second sequence
    • DP table construction: O(m × n)
    • LCS reconstruction: O(m + n)
  • Space Complexity: O(m × n)
    • Stores DP table of size (m+1) × (n+1)
    • Additional space for backtracking

Key Steps:

  • Table Construction:
    • Build DP table bottom-up
    • Compare characters and fill table entries
    • Use previous results for optimal subproblems
  • LCS Construction:
    • Backtrack through the DP table
    • Follow matches to construct LCS
    • Start from bottom-right corner

Example:

Sequence 1: "ABCDGH"

Sequence 2: "AEDFHR"

Longest Common Subsequence: "ADH"

Explanation: Characters 'A', 'D', and 'H' appear in both sequences in the same relative order.
